# Comprehensive PINN Analysis Dashboard

A rigorous visualization suite for dissertation-quality analysis of Physics-Informed Neural Network (PINN) financial forecasting models.

## Overview

This dashboard provides comprehensive visualizations across **6 key categories**:

| # | Category | Purpose | Key Visualizations |
|---|----------|---------|-------------------|
| 1 | **Predictive Performance** | Training and prediction quality | Loss curves, residual analysis, QQ plots |
| 2 | **Economic Effectiveness** | Trading performance | Cumulative returns, Sharpe ratios, drawdowns |
| 3 | **Risk Diagnostics** | Risk assessment | VaR/CVaR, volatility analysis, stress tests |
| 4 | **Comparative Evaluation** | Model comparison | Radar charts, heatmaps, Diebold-Mariano tests |
| 5 | **Explainability** | Model interpretability | Physics constraints, feature sensitivity |
| 6 | **Regime Robustness** | Market regime analysis | Regime performance, correlations, stability |

## Quick Start

### 1. Generate Analysis Data

Before running the dashboard, generate predictions and analysis data:

```bash
# Generate data for all models
python generate_analysis_data.py --all

# Or for a specific model
python generate_analysis_data.py --model pinn_gbm
```

### 2. Run the Dashboard

**Option A: Standalone**
```bash
streamlit run src/web/comprehensive_analysis_dashboard.py
```

**Option B: From main app**
```bash
streamlit run src/web/app.py
# Navigate to "Comprehensive Analysis" in sidebar
```

## Interpretation Guide

### Key Rules

| Visualization | What It Tells You |
|--------------|------------------|
| Loss curves | If the model trains properly |
| Return/Drawdown plots | If the model makes money |
| Risk plots | If the model survives stress |
| Explainability plots | If the model makes economic sense |

### Warning Signs

- **Diverging train/val loss**: Overfitting
- **Wildly unstable rolling Sharpe**: Luck, not skill
- **High volatility during calm periods**: Model increases risk when it shouldn't
- **VaR breach rate >> 5%**: Poor risk modeling
- **Random features important**: Overfitting to noise
- **Only works in bull markets**: Regime-dependent, not production ready

## Troubleshooting

### No Data Available

```
Warning: No model results found
```

**Solution:** Run model evaluations first:
```bash
python compute_all_financial_metrics.py
```

### No Training History

```
Warning: No training history found for {model}
```

**Solution:** Training history is saved automatically during training. Retrain the model or check `Models/` directory.

### No Predictions

```
Info: Using synthetic predictions based on model metrics
```

**Solution:** Generate real predictions:
```bash
python generate_analysis_data.py --all
```
